What is Quant72?

Quant72 is a DeFi agent network for Solana — an MCP server that wraps the Solana Agent Kit so any AI model can read the market and act on-chain through plain-English conversation.

Under the hood, Quant72 exposes a curated set of MCP actions covering live prices, ML price inference, trending radar, narrative analysis, smart-money sentiment, token risk scoring, wallet operations and liquidity provisioning. Point Claude, Cursor or any MCP-compatible client at the server and your agent gains a full on-chain market toolkit.

In one line

The DeFi Agent Network to Empower On-chain Trading — data powered by Hubble.

What you can build

  • A conversational trading copilot (see the Terminal)
  • Autonomous agents that monitor narratives & trending flows
  • Risk-screening bots that score tokens before execution
  • Portfolio assistants that read balances and move assets

# Installation

Quant72's MCP server is built on solana-agent-kit. Install the core kit, then scaffold the server with the actions you want to expose.

1 · Install dependencies

terminal
# core kit + dotenv
npm install solana-agent-kit dotenv

2 · Configure environment

Create a .env with your keys (see API & Env Config).

.env
SOLANA_PRIVATE_KEY=your_base58_private_key
RPC_URL=https://api.mainnet-beta.solana.com
OPENAI_API_KEY=sk-...
ELFA_API_KEY=...        # smart mentions
COINGECKO_PRO_API_KEY=... # trending / gainers

3 · Create the MCP server

This is the Quant72 server definition — register the actions your strategy needs:

quant72-mcp.ts
import { ACTIONS, SolanaAgentKit, startMcpServer } from "solana-agent-kit";
import * as dotenv from "dotenv";
dotenv.config();

const agent = new SolanaAgentKit(
  process.env.SOLANA_PRIVATE_KEY,
  process.env.RPC_URL,
  { OPENAI_API_KEY: process.env.OPENAI_API_KEY || "" },
);

const mcp_actions = {
  // market data
  FETCH_PRICE:          ACTIONS.FETCH_PRICE_ACTION,
  GET_PRICE_INFERENCE:  ACTIONS.GET_PRICE_INFERENCE_ACTION,
  GET_ALL_TOPICS:       ACTIONS.GET_ALL_TOPICS_ACTION,
  GET_ASSET:            ACTIONS.GET_ASSET_ACTION,
  // trending
  TRENDING_TOKENS:      ACTIONS.GET_COINGECKO_TRENDING_TOKENS_ACTION,
  TRENDING_POOLS:       ACTIONS.GET_COINGECKO_TRENDING_POOLS_ACTION,
  TOP_GAINERS:          ACTIONS.GET_COINGECKO_TOP_GAINERS_ACTION,
  // social / narratives
  SMART_MENTIONS:       ACTIONS.ELFA_GET_SMART_MENTIONS_ACTION,
  MENTIONS_BY_TICKER:   ACTIONS.ELFA_GET_TOP_MENTIONS_BY_TICKER_ACTION,
  // wallet
  BALANCE:              ACTIONS.BALANCE_ACTION,
  TOKEN_BALANCES:       ACTIONS.TOKEN_BALANCES_ACTION,
  TRANSFER:             ACTIONS.TRANSFER_ACTION,
};

startMcpServer(mcp_actions as any, agent, {
  name: "quant72-mcp", version: "0.1.0",
});

4 · Run it

terminal
npx tsx quant72-mcp.ts

# Quick Start — connect Claude & Cursor

Quant72 speaks the Model Context Protocol, so any MCP client can drive it. Register the server in your client config.

Claude Desktop

Add Quant72 to claude_desktop_config.json:

claude_desktop_config.json
{
  "mcpServers": {
    "quant72": {
      "command": "npx",
      "args": ["tsx", "/path/to/quant72-mcp.ts"],
      "env": { "RPC_URL": "https://api.mainnet-beta.solana.com" }
    }
  }
}

Cursor

Open Settings → MCP → Add Server, point it at the same command, and restart. Quant72's tools appear in the tool picker automatically.

Try it

Once connected, ask: "what's trending on Solana and which narratives are heating up?" — or skip setup and explore the hosted Quant72 Terminal.

# API & Env Config

Quant72 reads its configuration from environment variables. Only the first three are required for core functionality; the rest unlock specific tool groups.

VariableRequiredUsed by
SOLANA_PRIVATE_KEYYesSigning — wallet & transfers
RPC_URLYesAll on-chain reads/writes
OPENAI_API_KEYYes*Agent reasoning
ELFA_API_KEYNoSmart mentions, sentiment
COINGECKO_PRO_API_KEYNoTrending, gainers, pools
ALLORA_API_KEYNoPrice inference, topics
Security

Quant72 signs and sends transactions with the provided wallet. Run it in a secure environment and never commit your private key or share it.

# FAQ

Is Quant72 custodial?

No. The server runs locally with your own keypair and RPC. Your keys never leave your environment.

Which models work?

Any MCP-capable client — Claude Desktop, Cursor, or a custom agent built on the Vercel AI SDK / LangChain. The kit is model-agnostic.

Does the hosted Terminal trade real funds?

The Terminal demo uses illustrative mock data and does not execute on-chain transactions. Run your own MCP server to go live.

Where does the data come from?

Market data is aggregated via Hubble, with CoinGecko, Allora and ELFA powering trends, inference and sentiment respectively.

Is this financial advice?

No. Quant72 surfaces data and model outputs to inform your own decisions. Always do your own research.
